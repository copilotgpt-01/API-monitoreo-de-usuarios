Documentación API monitoreo de usuarios

Esta documentación describe los endpoints disponibles en la API de Eventos, permitiendo a los organizadores gestionar eventos y a los usuarios inscribirse en ellos. Todos los endpoints protegidos requieren autenticación mediante un token JWT.

Autenticación

Para acceder a los endpoints protegidos, es necesario incluir en la solicitud HTTP un header de autorización con el token JWT obtenido durante el proceso de login. El token debe ser incluido en los Headers de la siguiente manera: 

key= "Authorization" value = "TU_TOKEN_JWT_AQUÍ"

De igual manera solo los usuarios con el rol de organizador pueden acceder a los endpoints protegidos.

GET http://localhost:3000/api/v1/event/:eventId/users

Descripción: Permite a los organizadores ver la lista de usuarios registrados en un evento específico. Requiere autenticación.

Parámetros:

  • eventId: ID del evento.

Respuesta:

{
    "evento": "Nombre del Evento",
    "usuarios": [
        {
            "usuarioNombre": "Nombre del Usuario",
            "email": "Email del Usuario"
        }
    ],
    "total": 1
}

PUT http://localhost:3000/api/v1/event/:eventId/users/:userId

Descripción: Permite a los organizadores modificar el estado de inscripción de un usuario en un evento. Requiere autenticación.

Parámetros:

  • eventId: ID del evento.
  • userId: ID del usuario.
  • nuevoEstado: Nuevo estado de inscripción. (pendiente, confirmado, cancelado, etc.).

Respuesta:

{
    "mensaje": "El estado de inscripción ha sido actualizado."
}
